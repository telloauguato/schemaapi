### Introdução

Bem-vindo à documentação do **SchemaAPI**, uma ferramenta poderosa e versátil projetada para gerar dados fictícios para protótipos de sistemas. O SchemaAPI permite que desenvolvedores criem dados realistas e personalizados para testar e validar seus aplicativos durante o desenvolvimento.

#### O que é o SchemaAPI?

O SchemaAPI é um serviço RESTful que oferece a geração de dados fictícios configuráveis, ajudando os desenvolvedores a criar rapidamente conjuntos de dados realistas para suas aplicações. Seja para testes unitários, protótipos de UI/UX, ou simulações de cargas de trabalho, o SchemaAPI pode fornecer os dados necessários de forma rápida e eficiente.

#### Principais Características

- **Geração Personalizada de Dados**: Defina esquemas personalizados para gerar dados que atendam às suas necessidades específicas.
- **Suporte a Múltiplos Tipos de Dados**: Geração de uma ampla variedade de tipos de dados, incluindo nomes, endereços, números, datas e muito mais.
- **API Simples e Intuitiva**: Interface RESTful fácil de usar, facilitando a integração com qualquer aplicação.
- **Alta Performance**: Capacidade de gerar grandes volumes de dados de forma rápida.
- **Configuração Flexível**: Ajuste os parâmetros e padrões dos dados conforme necessário.

#### Caso de Uso

Imagine que você está desenvolvendo um novo aplicativo de gerenciamento de clientes e precisa de dados realistas para testar o sistema. Com o SchemaAPI, você pode definir um esquema para gerar nomes de clientes, endereços, números de telefone, e até mesmo informações de pedidos e transações. Isso permite que você valide a funcionalidade do seu aplicativo e identifique problemas antes de ir para produção.

#### Exemplo Rápido

Aqui está um exemplo simples de como utilizar o SchemaAPI para gerar dados fictícios de usuários:

1. **Definir o Esquema**:

    ```json
    {
      "type": "object",
      "properties": {
        "firstName": {
          "type": "string",
          "faker": "name.firstName"
        },
        "lastName": {
          "type": "string",
          "faker": "name.lastName"
        },
        "email": {
          "type": "string",
          "faker": "internet.email"
        },
        "address": {
          "type": "object",
          "properties": {
            "street": {
              "type": "string",
              "faker": "address.streetAddress"
            },
            "city": {
              "type": "string",
              "faker": "address.city"
            },
            "zipcode": {
              "type": "string",
              "faker": "address.zipCode"
            }
          }
        }
      },
      "required": ["firstName", "lastName", "email", "address"]
    }
    ```

2. **Fazer uma Solicitação para a API**:

    ```sh
    curl -X POST https://api.schemaapi.com/generate \
    -H "Content-Type: application/json" \
    -d '{
      "schema": {
        "type": "object",
        "properties": {
          "firstName": { "type": "string", "faker": "name.firstName" },
          "lastName": { "type": "string", "faker": "name.lastName" },
          "email": { "type": "string", "faker": "internet.email" },
          "address": {
            "type": "object",
            "properties": {
              "street": { "type": "string", "faker": "address.streetAddress" },
              "city": { "type": "string", "faker": "address.city" },
              "zipcode": { "type": "string", "faker": "address.zipCode" }
            }
          }
        },
        "required": ["firstName", "lastName", "email", "address"]
      }
    }'
    ```

3. **Receber Dados Gerados**:

    ```json
    {
      "firstName": "John",
      "lastName": "Doe",
      "email": "john.doe@example.com",
      "address": {
        "street": "123 Main St",
        "city": "Anytown",
        "zipcode": "12345"
      }
    }
    ```

Com essas etapas simples, você pode gerar dados fictícios personalizados para usar em seus testes e desenvolvimento. O SchemaAPI facilita a criação de dados realistas, permitindo que você se concentre no desenvolvimento de funcionalidades e na melhoria da qualidade do seu software.
