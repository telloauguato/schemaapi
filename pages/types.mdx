# Types

The types follow a pattern where each type will have a `key` that represents the name of the object to be displayed when the data is generated and a `type` that indicates the type of data expected. These are mandatory.

```json showLineNumbers copy filename="default body"
{
    "key": "<nameOfKey>",
    "type": "<typeOfData>"
}
```

## Name

Type 'name' generates a dummy name. This is useful for creating names of fictitious people or entities in your tests or prototypes.

```json showLineNumbers copy filename="name type"
{
    "key": "name",
    "type": "name",
    "prefix": "",
    "sufix": ""
}
```

## Value

The 'value' type allows you to define a predefined value that will be returned when the dummy data is generated. It's useful when you need a specific value instead of random data.

```json showLineNumbers copy filename="value type"
{
    "key": "value",
    "type": "value",
    "value": "user defined value",
    "prefix": "",
    "sufix": ""
}
```

## username

The 'username' type generates a fictitious username. It is commonly used in scenarios where you need fictitious usernames to simulate user interactions.

```json showLineNumbers copy filename="username type"
{
    "key": "username",
    "type": "username",
    "prefix": "",
    "sufix": ""
}
```

## Pattern

The 'pattern' type allows you to define a custom pattern to generate dummy data. You can specify the desired pattern, including special characters to represent random values.

```json showLineNumbers copy filename="pattern type"
{
    "key": "pattern",
    "type": "pattern",
    "pattern": "XXXX-XXXX", 
    "prefix": "",
    "sufix": ""
}
```

## Options

The 'options' type allows you to choose an option from a set of defined options. This is useful when you want dummy data to be chosen from a specific set of values.

```json showLineNumbers copy filename="options type"
{
    "key": "options",
    "type": "options",
    "length": 2,
    "options": ["Option 1", "Option 2", "Option 3"]
}
```

## Location Types ('street'|'city'|'country'|'state')

These types generate fictitious location-related data, such as street addresses, city names, countries, and states. They are useful for simulating contact and location information.

```json showLineNumbers copy filename="street|city|country|state type"
{
    "key": "<iden>",
    "type": "street|city|country|state",
    "prefix": "",
    "sufix": ""
}
```

## Schema

The 'schema' type allows you to create custom built-in types nested within your specification. You can define built-in types and your own settings.

```json showLineNumbers copy filename="schema type"
{
    "key": "schema",
    "type": "schema",
    "schema": {
        "name": "User",
        "length": -5,
        "content": [
            {
                "key": "username",
                "type": "username",
                "prefix": "@"
            },
            {
                "key": "active",
                "type": "options",
                "options": [
                    true,
                    false
                ]
            }
        ]
    }
}
```
