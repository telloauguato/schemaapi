# Get Started

**SchemaAPI** is a powerful tool for developers in need of realistic and accurate fictitious data for testing, prototyping, or any other purpose. With SchemaAPI, you can generate a wide range of fictitious data quickly and easily, saving you time and resources in your project development.

## Key Features

- Name Generation: Need fictitious names of individuals? SchemaAPI can provide a variety of realistic names to enrich your tests.

- nteger Numbers: Generate random integer numbers with ease by configuring the desired ranges.

- Usernames: Create unique usernames for your applications with just a few clicks.

- Addresses: Obtain fictitious addresses, including streets, cities, states, and countries, to simulate real-world environments.

- Contact Data: Add contact information such as email addresses and phone numbers to your tests conveniently.

- Profile Images: Integrate fictitious avatars and profile images into your applications to enhance the visual experience.

- Custom Data: Personalize data generation to meet your specific testing needs.

## How to Use

Using SchemaAPI is straightforward:

1. Create a public repository on Github.

2. Set `main` as your default branch.

3. Create the schema file: `*.schema.json`.

4. Now just request the schema with the url: `https://schemaapi.vercel.app/api/<repository>@<username>/<pathAndNameOfSchemaFile>`. No need to append `.schema.json`
